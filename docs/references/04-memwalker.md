# MemWalker — Interactive Reading via Tree Navigation

> **相关度: ★★★★☆** — LLM 主动导航决定"看什么"（导航直觉最接近）

## 基本信息

- **标题**: Walking Down the Memory Maze: Beyond Context Limit through Interactive Reading
- **作者**: Howard Chen, Ramakanth Pasunuru, Jason Weston, Asli Celikyilmaz
- **日期**: 2023-10
- **链接**: https://arxiv.org/abs/2310.05029
- **领域**: cs.CL

## 核心方法

1. **预处理**: 将长上下文文本构建为摘要节点树
2. **交互式导航**: 收到查询后，LLM 作为**交互式智能体**在树上导航
3. **迭代探索**: LLM 一步步决定走哪个分支，收集足够信息后回答
4. **可解释性**: 展示推理步骤和相关文本片段

## 与语义路标的对比

| 维度 | MemWalker | 语义路标 |
|------|-----------|---------|
| **核心直觉** | LLM 自主决定"看什么" | LLM 自主决定"走哪条边" |
| **处理对象** | 单篇长文档的上下文压缩 | **跨文档的知识组织** |
| **结构** | 树 | **网**（多归属 + 跨领域边） |
| **自生长** | ❌ 无（静态构建） | ✅ LLM 自主分裂/合并/建边 |
| **边关系** | ❌ 无边概念 | ✅ 自然语言边描述 |
| **增量更新** | ❌ 无（一次性处理） | ✅ 增量入库 + 重整 |
| **导航者** | LLM | LLM |

## 重叠点（核心直觉一致）

- **"LLM 自主决定导航方向"** — 这是最本质的共同点 ✅
- 把长内容构建成结构，LLM 在结构上主动导航 ✅
- 交互式信息收集（不是一次性全量灌入）✅
- 增强可解释性 ✅

## 关键差异

1. MemWalker 处理**单篇长文档**，语义路标处理**持续增长的知识库**
2. MemWalker **没有知识组织**的概念，只是上下文压缩手段
3. MemWalker 是**树搜索**（从上往下），语义路标是**图探索**（可跳转、回溯、发现意外关联）
4. MemWalker 没有自生长、没有边关系、没有多归属
