# RAPTOR — Recursive Abstractive Processing for Tree-Organized Retrieval

> **相关度: ★★★★☆** — 递归摘要构建层次树 + 树上检索

## 基本信息

- **标题**: RAPTOR: Recursive Abstractive Processing for Tree-Organized Retrieval
- **作者**: Parth Sarthi, Salman Abdullah, Aditi Tuli, Shubh Khanna, Anna Goldie, Christopher D. Manning
- **日期**: 2024-01 (ICLR 2024)
- **链接**: https://arxiv.org/abs/2401.18059
- **领域**: cs.CL
- **评估结果**: QuALITY 基准 +20% 绝对准确率 (w/ GPT-4)

## 核心方法

1. **文本分块**: 长文档切分为小文本块
2. **递归处理**: 嵌入 → 聚类（软聚类）→ LLM 摘要 → 循环
3. **树形构建**: 原始文本为叶节点，摘要为父节点，自底向上逐层构建
4. **树状检索**: 推理时遍历整棵树，同时获取细节（下层）和全局信息（上层）

## 与语义路标的对比

| 维度 | RAPTOR | 语义路标 |
|------|--------|---------|
| **构建方向** | 自底向上（聚类 + 摘要） | **增量入库 + 定期重整**（自生长） |
| **检索方式** | embedding 相似度遍历树 | **LLM 推理游走图** |
| **拓扑结构** | 树（静态构建） | 网（动态自生长） |
| **匹配决策者** | 系统（余弦相似度） | **LLM** |
| **摘要/路标** | ✅ LLM 生成摘要节点 | ✅ LLM 生成路标描述 |
| **增量更新** | 需重建整棵树 | 增量入库 + 惰性重整 |
| **多归属** | ✅ 软聚类支持 | ✅ 网状天然支持 |

## 重叠点（高度相似）

- 用 LLM 递归生成摘要 → 构建多层结构 ✅（"摘要节点"≈"路标描述"）
- 检索时在结构上导航 ✅
- 多层抽象整合信息 ✅

## 关键差异

1. RAPTOR 是**静态一次性构建**，语义路标是**动态增量自生长**
2. RAPTOR 用 **embedding + 向量检索**做匹配，语义路标用 **LLM 推理**
3. RAPTOR 处理**单文档/语料**的离线索引，语义路标处理**持续流入的增量数据**
4. RAPTOR 是**树**（层次分类），语义路标是**网**（关系网络，支持跨领域发现）
