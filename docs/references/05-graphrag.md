# GraphRAG — Graph RAG for Query-Focused Summarization

> **相关度: ★★★☆☆** — 图结构组织 + 社区摘要多层抽象

## 基本信息

- **标题**: From Local to Global: A Graph RAG Approach to Query-Focused Summarization
- **作者**: Darren Edge, Ha Trinh, Newman Cheng 等 (10人)
- **日期**: 2024-04 (Microsoft Research)
- **链接**: https://arxiv.org/abs/2404.16130
- **领域**: cs.CL, cs.AI, cs.IR

## 核心方法

1. **实体知识图谱构建**: LLM 从源文档中提取实体（节点）+ 关系（边）
2. **社区检测**: Leiden 算法识别紧密相关的实体群组
3. **社区摘要**: LLM 为每个社区预生成描述性摘要（离线）
4. **查询处理**: 利用社区摘要生成部分响应 → 汇总为最终回答

## 与语义路标的对比

| 维度 | GraphRAG | 语义路标 |
|------|----------|---------|
| **图的节点** | 实体（人/事/物） | **概念领域 + 自然语言路标** |
| **图的边** | 关系提取（实体间） | **语义关联描述**（自然语言） |
| **构建方式** | LLM 提取 → 系统社区检测 | **LLM 自主构建 + 重整** |
| **检索方式** | 向量匹配社区摘要 | **LLM 推理导航** |
| **多层抽象** | ✅ 社区层级 | ✅ 语义网层级 |
| **自生长** | ❌ 预定义 schema | ✅ LLM 自主演化 |
| **增量更新** | 需重建图谱 | 增量入库 + 惰性重整 |

## 重叠点

- 图结构组织知识 ✅
- LLM 从文本中提取关系 ✅
- 多层抽象（社区摘要 ≈ 路标描述）✅

## 关键差异

1. GraphRAG 的节点是**具体实体**，语义路标的节点是**概念领域**
2. GraphRAG 仍用**向量匹配**做检索，语义路标用 **LLM 导航**
3. GraphRAG 需要**预定义 schema**（实体类型、关系类型），语义路标**完全由 LLM 自由定义**
4. GraphRAG 的图谱构建是**离线一次性**的，语义路标是**在线增量自生长**的
