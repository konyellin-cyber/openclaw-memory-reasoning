# 语义路标记忆系统 — 项目愿景与定位

> 用 100% 的你去匹配整个世界

**日期**: 2026-03-05  
**参与**: 作者 + AI 分身

---

## 一、项目是什么

一个运行在 OpenClaw 上的语义路标记忆系统。

分身（林风过竹）不只记得你说过什么（内部记忆），还能看到外面发生了什么（外部记忆），然后主动把对你有价值的信息推荐给你。

**核心公式**：

```
推荐 = f(你是谁, 你在想什么, 外面发生了什么)
     = f(内部记忆 × 外部记忆)
```

## 二、为什么要做这个

### 思路演变

| 阶段 | 方向 | 问题 |
|------|------|------|
| 2/21 | 双塔向量检索（外挂记忆召回系统） | 补偿层 — 模型长上下文能力变强后被淘汰 |
| 2/28 上午 | 轻量模型做记忆路由推理 | 也是补偿层 — 模型变强后自己能路由 |
| 2/28 下午 | 个人分身推荐系统 | 模型越强推荐越准 — 壁垒在记忆积累和信息源品味 |
| **3/5** | **语义路标记忆系统** | **内部记忆重整驱动外部记忆检索与淘汰** |

### 核心判断

向量检索、记忆路由都是"补偿模型暂时做不好的事"。
语义路标记忆系统是"模型再强也替代不了的系统能力"——
因为模型没有持久状态、没有外部信息源、不会主动找你。

**3/5 新增判断**：内部记忆的准确重整是一切的基础。内部记忆越准确，生成的检索 query 越精准，外部记忆的淘汰越合理。内部记忆不是附属品，是驱动力。

### 为什么现在做

- 自己就是用户，闭环验证，反馈环路极短
- Inner Memory 已经积累了（OpenClaw workspace）
- 推荐系统工作经验可直接迁移
- 完全个人项目，零公司依赖
- 周末就能跑通最小切片

## 三、与传统推荐系统的同构映射

| 概念 | 公众号推荐（工作） | 本项目 |
|------|-------------------|--------|
| 用户画像 | 行为特征 + 标签 | **完整个人记忆 + 价值观** |
| 物品池 | 公众号文章 | arXiv 论文 + RSS 内容 |
| 召回 | 多路召回 → 粗筛 | 内部/外部双源融合 |
| 排序 | CTR/CVR 预估 | **"对你的价值"预估** |
| 多目标 | 点击 × 时长 × 互动 | 相关性 × 新颖性 × 时效性 × 行动力 |
| 探索/利用 | 兴趣探索 vs 确定性推荐 | "你不知道但应该知道" vs "你正在关注的" |
| 冷启动 | 新用户 / 新内容 | 新信息源 / 新兴趣领域 |

传统推荐系统是"用 1% 的你去匹配 100% 的内容"。
这个系统是"用 100% 的你去匹配整个世界"。

## 四、双记忆模型

### 内部记忆（Inner Memory）

来源：OpenClaw `workspace/memory/`

包含：个人记忆、价值观、偏好、决策历史、思维方式、项目上下文、当前关注点、洞察与方法论。

**定位**：需求侧 — 驱动检索、驱动淘汰。重整内部记忆不是为了"管理日记"，而是让系统更好地理解用户关心什么、在做什么。

### 外部记忆（Outer Knowledge）

来源：arXiv RSS、技术博客、行业媒体等公开信息源。

**定位**：供给侧 — 通过搜索持续扩充，通过淘汰控制规模。

### 内外关系

```
内部记忆（重整后） → 生成更精准的查询意图 → 检索外部记忆（论文库）
                                              ↑
                                        搜索到的论文作为
                                        外部索引的增量来源
                                              ↓
                                        淘汰机制控制规模
```

**不是对称的两套系统，是内部驱动外部的关系。**

## 五、产品形态

直接在 OpenClaw 上实现，不做额外 app/bot。

### 对话中自然推荐

聊天过程中，agent 自己判断当前话题和 feed 中的论文相关，主动调用工具查找，在回复中自然带出推荐。没有任何外挂判断层。

### 反馈闭环

- 你点开/忽略的论文 → 记录偏好
- 你的评价 → 调整推荐
- 随时间积累，推荐越来越准

## 六、设计原则（铁律）

| 原则 | 说明 |
|------|------|
| **一切基于 LLM 基座** | 不做外挂检索/向量召回/路由层/触发判断层，agent 自主推理和调用工具 |
| **不做补偿层** | 如果一个能力属于"模型能力的子集"，模型变强后就会淘汰它——不投资 |
| **内部记忆驱动外部记忆** | 内部记忆是需求侧，外部记忆是供给侧，不是对称的两套系统 |
| **同一套重整机制** | 内外记忆用同样的语义路标方式重整，共享引擎，隔离数据 |
| **个人项目定位** | 不依赖任何私有数据或内部接口 |
| **公开数据源** | 所有数据通过公开合法渠道获取（RSS） |
| **产品形态用 OpenClaw** | 不做额外 app/bot/CLI |
| **渐进式** | 先跑通一个信息源，再扩展 |

**判断标准**：设计任何新功能前先问——"这件事模型足够强后能自己做吗？"
- 能 → 不做（补偿层）
- 不能 → 做（系统能力：持久化、定时拉取、外部数据源、反馈记录、淘汰调度）
